# 🔧 План: Как наверстать упущенное (Интеграция фич в код Адели)

Ты решил поработать головой и реализовать упущенные фишки самостоятельно — это отличный подход! Ниже пошаговый план того, что тебе нужно написать в Django, чтобы вернуть функционал для Кабинета (Dashboard), Авторизации и Отзывов/Избранного.

---

## Шаг 1: Восстановить статистику (`/api/stats/`)
Фронтенд сейчас ожидает получить 4 цифры при открытии кабинета.

**Что делать:**
1. Открой `petcare-backend/shelters/views.py` (или `animals/views.py`).
2. Добавь импорт декоратора: `from rest_framework.decorators import api_view` и `from rest_framework.response import Response`.
3. Создай функцию `api_stats(request)`, которая будет считать объекты:
   ```python
   from animals.models import Pet
   from shelters.models import Shelter
   
   @api_view(['GET'])
   def get_stats(request):
       total_shelters = Shelter.objects.count()
       total_pets = Pet.objects.count()
       adopted_pets = Pet.objects.filter(is_adopted=True).count()
       
       return Response({
           "shelters_count": total_shelters,
           "pets_count": total_pets,
           "adopted_count": adopted_pets,
           "total_pets": total_pets
       })
   ```
4. В файле `petcare-backend/core/urls.py` добавь путь к этой функции: `path('api/stats/', get_stats)`.

## Шаг 2: Модель "Избранного" (Favorites)
Чтобы пользователи могли добавлять животных в избранное, нужна таблица, которая свяжет Пользователя (User) и Питомца (Pet).

**Что делать:**
1. Открой `petcare-backend/animals/models.py`.
2. Импортируй стандартного пользователя: `from django.contrib.auth.models import User`.
3. Создай модель `Favorite`:
   ```python
   class Favorite(models.Model):
       user = models.ForeignKey(User, on_delete=models.CASCADE)
       pet = models.ForeignKey(Pet, on_delete=models.CASCADE)
       created_at = models.DateTimeField(auto_now_add=True)
       
       class Meta:
           unique_together = ('user', 'pet') # Чтобы нельзя было лайкнуть дважды
   ```
4. После создания модели не забудь запустить `python manage.py makemigrations` и `python manage.py migrate`.
5. Напиши View (представление), которое будет принимать `POST`-запросы для добавления/удаления лайка и отдавать список лайкнутых в ответ на `GET`.

## Шаг 3: Аутентификация и пользователи (JWT Token)
Сейчас в коде Адели любой человек в интернете имеет доступ ко всему. Чтобы заработал логин/регистрация на фронтенде, нужно вернуть поддержку JWT токенов.

**Что делать:**
1. Нужно установить библиотеку: `pip install djangorestframework-simplejwt`.
2. В `core/settings.py` в блоке `REST_FRAMEWORK` указать `JWTAuthentication`.
3. В `core/urls.py` импортировать готовые роуты:
   ```python
   from rest_framework_simplejwt.views import TokenObtainPairView, TokenRefreshView
   
   urlpatterns = [
       ...
       path('api/auth/login/', TokenObtainPairView.as_view(), name='token_obtain_pair'),
       path('api/auth/refresh/', TokenRefreshView.as_view(), name='token_refresh'),
   ]
   ```
4. Для Регистрации придется написать небольшую свою `APIView`, которая будет принимать `username`, `email` и `password`, и делать `User.objects.create_user(...)`.

## Шаг 4: Контакты (Форма на главной)
Пользователь заполняет Имя, Телефон и Сообщение на Главной странице сайта.

**Что делать:**
1. В `shelters/models.py` добавь модель `ContactMessage` (поля name, phone, message).
2. Создай простенький ModelSerializer в `serializers.py` для этой модели.
3. В `views.py` добавь класс-view: `class ContactMessageCreate(...)`.
4. Зарегистрируй роут `/api/contact/` в `urls.py`.
