### @if and @for (Angular 17+ syntax)

This is Angular's **new built-in template syntax** (replacing the old `*ngIf` and `*ngFor` directives).

| Directive | Explanation | Example in code |
|---|---|---|
| `@for` | A loop — iterates over an array and renders an HTML element for each item. `track` helps Angular identify which element belongs to which item so it doesn't re-render everything on update. | `@for (pet of filteredPets; track pet.id)` (in `pet-list.component.html`) renders an HTML card for each pet. |
| `@if` | A condition — renders a block only when the expression is true. Used to show different UI depending on app state. | `@if (auth.isLoggedIn())` (in `navbar.component.html`) shows "Dashboard / Sign Out" if logged in, otherwise "Sign In". |

### JWT Authentication

**JWT (JSON Web Token)** is a token the server issues upon login. It is stored in `localStorage` and attached to every request so the backend knows who is making the request.

**`auth.service.ts`** — manages the token lifecycle:
- `login()` — receives `access` and `refresh` tokens from Django, saves them to `localStorage`
- `logout()` — removes tokens, resets state (`currentUser`, `isLoggedIn`)
- `isLoggedIn` / `currentUser` — Angular Signals that reactively update the UI when auth state changes

**`auth.interceptor.ts`** — **automatically** appends a header to every outgoing HTTP request:
```
Authorization: Bearer <token from localStorage>
```
This happens transparently — components are unaware of it, the interceptor works like middleware.

### 1 Angular Service using HttpClient
**`pet.service.ts`** is the core bridge between our visual UI and backend. 

- **Singleton (`@Injectable({ providedIn: 'root' })`)**: Angular creates only one shared instance of this service, saving memory.

### Error Handling

**`error.interceptor.ts`** — **global HTTP error handler**. Intercepts all error responses via `catchError`:

| HTTP Status | What happens |
|---|---|
| `401 Unauthorized` | Token expired — removes it from localStorage and redirects to `/login` |
| `403 Forbidden` | No access rights — shows "Access denied" message |
| `404 Not Found` | Resource does not exist |
| `0` (network error) | Backend unreachable — "Check your connection" |
| `500+` | Internal server error |

**`login.component.ts`** — **local error handling** directly in the component:
- Inside `subscribe({ error: (err) => {...} })` it parses the Django error response
- Writes the error text to the `errorMessage` Signal
- The template automatically renders `@if (errorMessage()) { <div class="alert">... }` without a page reload

# Slide 3: Backend

## Backend Requirements (Django REST Framework)

### 🧱 Database & Models
- **4 Models**: `Pet`, `Shelter`, `Favorite` (custom logic), and `User` (built-in auth).
- **1 Custom Model Manager**: `PetManager` handles custom queryset logic (e.g., filtering available pets).
- **2 ForeignKey Relations**: 
  - `Shelter` ➔ `Pet` (One-to-Many).
  - `User` ➔ `Favorite` (Links the object to the authenticated user).

### 🛠 DRF Components
- **2 `serializers.Serializer`**: `LoginSerializer`, `ContactMessageSerializer` (Handles non-model validation).
- **2 `serializers.ModelSerializer`**: `PetSerializer`, `ShelterSerializer` (Transforms Python models to JSON).
- **Function-Based Views (FBV)**: `api_stats` (Calculates lightweight total platform statistics).
- **Class-Based Views (CBV)**: `PetListAPIView` (Provides scalable GET endpoints with pagination/filtering).

### 🔄 CRUD Operations
Our API handles all four database operations:
- **C**reate: `RegisterView` (Registers a new user).
- **R**ead: `PetListAPIView` (Fetches the pet catalog).
- **U**pdate: `ProfileUpdateView` (Updates user details).
- **D**elete: `FavoriteDeleteView` (Removes a pet from favorites).

### 🔐 Security & Auth
- **Token-Based Authentication**: Implemented via `SimpleJWT` (`TokenObtainPairView`). Issues Access/Refresh tokens.
- **CORS Configuration**: `CORS_ALLOWED_ORIGINS` is configured to securely allow the Angular frontend port to request data.
